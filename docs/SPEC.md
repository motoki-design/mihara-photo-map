## 1. 目的

4/27現地踏査で青木氏が撮影した117枚の写真を、業務メモ・カテゴリ・場所と紐付けて地図上で閲覧できる状態にし、現地調査の事前学習・中間打合せの議論材料・市側への提示素材として活用する。

副次目的として、景観シミュレーション画像をBefore/After形式で同地点に重ねて閲覧できる仕組みを実装し、生成AI活用ワークフローを業務に取り込むためのモデルケースとする。

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 現地写真 | 4/27に青木氏が撮影した踏査写真（IMG_0001〜IMG_0117 の117枚） |
| 場所推定 | 写真ログMarkdownの「場所推定」テキスト。手がかりから推定された撮影位置の文字列 |
| 場所辞書 | 場所推定文字列のキーワードと緯度経度を対応付けた辞書（`data/location_dict.json`） |
| 景観シミュレーション | 生成AIで作成したBefore/After画像（A/B/C等のバリアント） |
| 写真ID | `IMG_NNNN.JPG` 形式のファイル名（PNG等の他拡張子も将来的に許容） |

### 3.2 詳細仕様

#### F1：地図表示
- ベース地図3種：
  - OpenStreetMap（**デフォルト**）
  - 地理院タイル（淡色）
  - 地理院タイル（標準）
- 切替UIは右上の Leaflet Layers Control（常時展開）
- 初期表示中心：[34.3974, 133.0792]、ズーム17
- 起動時に全ピンが収まる範囲に自動フィット

#### F2：写真ピン
- 117枚すべての写真をピン配置
- ピン色は推定精度別：
  - 緑（confirmed：辞書一致）
  - 橙（inherited：他写真からの参照継承）
  - 赤（interpolated/extrapolated：時系列補間/外挿）
  - 灰（fallback：推定不可、駅中心）
- シミュレーション画像が紐付いている写真のピンは**橙色の縁取り＋オレンジシャドウ**で強調

#### F3：ポップアップ
- 表示要素（上から順）：
  1. サムネ画像（クリックでモーダル拡大）
  2. ファイル名 ＋ 撮影時刻 ＋ 推定精度バッジ
  3. 場所
  4. 業務関連ポイント
  5. シミュレーション案（テキスト、写真ログから）
  6. カテゴリチップ
  7. 景観シミュレーション ▶（バリアント数だけボタン）

#### F4：写真モーダル
- 全画面オーバーレイ（黒背景92%）
- 中央に画像（最大幅100%、最大高100%、aspect維持）
- 下にキャプション：「ファイル名 ／ バリアントタイトル ／ 概要」
- 右上の × ボタンまたは画像クリックで閉じる

#### F5：踏査ルートライン
- 撮影時刻順に117点を結ぶポリライン
- スタイル：紺色 #1f3a68、weight 2、opacity 0.55、dashArray 5,5
- ツールバー「踏査ルート線」チェックで表示/非表示

#### F6：カテゴリフィルタ
- 9カテゴリ：
  - 照明 / 舗装 / サイン / 植栽 / ストリートファニチャー / 史跡 / 景観阻害物 / 建築 / 広場
- 各カテゴリの該当件数を `(N)` で表示
- 全選択／全解除ボタン
- 複数カテゴリ選択時は OR 条件（いずれかに該当すれば表示）

#### F7：サイドリスト
- 左幅300pxのサイドバー
- 撮影時刻順で全表示中の写真を一覧
- 各項目：サムネ60×45px ＋ ファイル名 ＋ 時刻 ＋ 場所 ＋ カテゴリバッジ
- シミュレーション付きには **SIM Nバッジ**（橙色）
- 検索ボックスでファイル名・場所・キーワードを部分一致絞り込み
- クリックで地図がその位置にズーム＋ポップアップ自動オープン
- ヘッダ「☰ 一覧」ボタンで開閉

#### F8：景観シミュレーション表示
- 写真にシミュレーション画像が紐付く場合、ポップアップに「▶ 景観シミュレーション N案」セクションを追加
- 各バリアントは独立ボタン（A：和洋中間モダン 等）
- ボタンクリックでモーダル展開、上部にタブを表示
- タブで「元写真／A／B／C」を即時切替（画像とキャプションが入れ替わる）

#### F9：シミュ付きのみフィルタ
- ツールバー「シミュ付きのみ」チェックで、シミュレーション画像が紐付いた写真のみを地図に表示
- 議論用の素材として「Before/After検討対象」を素早く把握する用途

#### F10：スマホ対応
- 768px以下のビューポートで：
  - サイドバーが画面外に隠れた状態で起動（☰ボタンで展開・ドロワー化）
  - ヘッダ情報が簡略化（meta部の文字を非表示）
  - タッチ操作に対応（Leaflet標準）

## 4. データ仕様

### 4.1 photos.geojson（GeoJSON FeatureCollection）

```json
{
  "type": "FeatureCollection",
  "features": [
    {
      "type": "Feature",
      "geometry": { "type": "Point", "coordinates": [133.07920, 34.39800] },
      "properties": {
        "file": "IMG_0001.JPG",
        "time": "14:57:24",
        "subject": "被写体テキスト...",
        "location_text": "場所推定の元テキスト...",
        "business_point": "業務関連ポイント...",
        "simulation_idea": "シミュレーション案テキスト...",
        "place_id": "station_concourse",
        "place_name": "三原駅構内コンコース",
        "accuracy": "confirmed",
        "categories": ["舗装", "サイン", "史跡"],
        "thumb": "assets/thumbs/IMG_0001.jpg",
        "photo": "assets/photos/IMG_0001.jpg",
        "simulations": []
      }
    }
  ]
}
```

`accuracy` の値域：`confirmed | inherited | interpolated | extrapolated | fallback`

`simulations` は配列。各要素：
```json
{
  "id": "A",
  "title": "和洋中間・モダンミニマル",
  "image": "assets/simulations/IMG_0009_A.jpg",
  "summary": "上下2段ディスク灯具..."
}
```

### 4.2 location_dict.json

```json
{
  "_meta": { "version": "0.1", "anchor": {...}, "walking_speed_kmh": 4.0 },
  "places": [
    {
      "id": "station_concourse",
      "name": "三原駅構内コンコース",
      "lat": 34.39800,
      "lon": 133.07920,
      "match": ["駅構内", "コンコース", "南北自由通路", ...]
    }
  ]
}
```

`match` の文字列が場所推定テキストに含まれる場合、その地点として確定。

### 4.3 simulations.json

```json
{
  "_meta": { "version": "0.1", "note": "..." },
  "IMG_0009.JPG": [
    {
      "id": "A",
      "title": "和洋中間・モダンミニマル",
      "image": "assets/simulations/IMG_0009_A.jpg",
      "summary": "..."
    },
    { "id": "B", ... },
    { "id": "C", ... }
  ],
  "IMG_0089.JPG": [ ... ]
}
```
