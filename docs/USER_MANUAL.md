# MiharaScape 使い方説明書

**MiharaScape** は、三原駅〜内港エリアで2026年4月27日に踏査した現地写真117枚と、景観シミュレーション結果を、地図上で閲覧できるWebアプリです。

🌐 **公開URL**: https://motoki-design.github.io/mihara-photo-map/

---

## 1. 動作環境

- **PC**: Chrome / Edge / Safari / Firefox（最新版推奨）
- **スマホ**: iOS Safari / Android Chrome
- **ネット接続**: 必要（地図タイル・写真は都度読み込み）
- **ログイン**: 不要

## 2. 起動

ブラウザのアドレスバーに次のURLを入力するだけ：

```
https://motoki-design.github.io/mihara-photo-map/
```

開くと、三原駅周辺の地図と117個のピンが自動表示されます。

## 3. 画面構成

```
┌──────────────────────────────────────────────────────┐
│ ヘッダ（紺色帯）                                       │
│ ☰一覧 / MiharaScape / サブタイトル                     │
├──────────────────────────────────────────────────────┤
│ ツールバー（薄ブルー帯）                               │
│ カテゴリ:[照明][舗装][サイン]... 全選択/全解除         │
│ ☑踏査ルート線 ☐シミュ付きのみ          表示: 117/117 │
├─────────────┬────────────────────────────────────────┤
│             │                                         │
│ サイドリスト │              地  図                    │
│ (検索ボックス│  ・ピン×117                           │
│  117枚一覧) │  ・踏査ルート線（破線）                 │
│             │  ・右上に地図切替コントロール           │
│             │                                         │
└─────────────┴────────────────────────────────────────┘
```

### 3.1 ヘッダ
- **☰ 一覧** ボタン：サイドリストの開閉
- **MiharaScape**：アプリ名
- 右側：撮影日時・撮影者などのメタ情報

### 3.2 ツールバー
- **カテゴリチップ**：9つのカテゴリ（照明・舗装・サイン・植栽・SF・史跡・景観阻害物・建築・広場）。チェックON/OFFで地図上のピンを絞り込む
- **全選択 / 全解除** ボタン：カテゴリの一括操作
- **☑ 踏査ルート線**：撮影順を結ぶ破線の表示/非表示
- **☐ シミュ付きのみ**：景観シミュレーションが紐付いた写真のみ表示
- **表示: N / 117**：現在地図に表示されている件数

### 3.3 サイドリスト（左側）
- 上部：🔍 検索ボックス（ファイル名・場所・キーワードで部分一致絞込）
- 各項目：サムネ＋ファイル名＋撮影時刻＋場所＋カテゴリバッジ
- **SIM N** バッジ（橙色）：シミュレーション画像が N 件紐付いた写真
- 項目クリック：地図がその位置にズーム＋ポップアップが自動オープン

### 3.4 地図
- **ベース地図**：右上のレイヤコントロールで切替可能
  - OpenStreetMap（デフォルト）
  - 地理院タイル（淡色）
  - 地理院タイル（標準）
- **ピン**：117個。ピン色は推定精度を表す（次項参照）
- **踏査ルート線**：紺色の破線、撮影時刻順に117点を結ぶ

### 3.5 ピンの色
| 色 | 推定精度 | 意味 |
|-----|----------|------|
| 🟢 緑 | confirmed | 場所辞書のキーワードと一致して位置を確定 |
| 🟠 橙（中色） | inherited | 「IMG_NNNN と同位置」のような参照で他写真の位置を継承 |
| 🔴 赤 | interpolated | 撮影時刻に基づいて前後の確定写真の間を線形補間 |
| ⚫ 灰 | fallback | 推定不可、駅中心に仮配置 |
| 🟧 橙縁取り＋影 | — | **景観シミュレーション画像が紐付いた写真**（精度色は内側の色で表現） |

## 4. 基本操作

### 4.1 ピンをクリックして写真を見る
1. 地図上のピンをクリック
2. ポップアップが開き、サムネ画像と次の情報が表示されます：
   - ファイル名 ／ 撮影時刻 ／ 推定精度バッジ
   - 場所
   - 業務関連ポイント
   - シミュレーション案（テキスト）
   - カテゴリ
   - **▶ 景観シミュレーション N案**（紐付いている場合）
3. サムネをクリックすると拡大モーダルが開く

### 4.2 写真を拡大表示
- ポップアップ内のサムネをクリック
- 黒背景の中央に1280px版が表示される
- 閉じる：右上の **×** ボタン または 画像の外側クリック

### 4.3 景観シミュレーションを比較
**例：IMG_0009（駅前広場・装飾洋風大型街路灯）**

1. 駅前広場の橙縁取りピン（IMG_0009）をクリック
2. ポップアップ最下部に **▶ 景観シミュレーション 3案** が表示される
3. ボタンをクリック：
   - `[A：和洋中間・モダンミニマル]`
   - `[B：和風行灯]`
   - `[C：街路灯撤去・低背分散照明への再編]`
4. モーダルが開き、上部にタブが表示される
5. タブをクリックして **元写真／A／B／C** を切替

タブ「元写真」は紺色、シミュレーションタブは橙色で表示されます。

### 4.4 カテゴリでフィルタ
- ツールバーのカテゴリチップでチェックを外すと、そのカテゴリのみのピンが地図から消える
- 複数チェックを外せば、AND ではなく OR で絞り込まれる（残ったカテゴリのいずれかに該当する写真が表示）
- **全選択 / 全解除** ボタンで一括切替

**用途例**：
- 照明だけON → 駅前広場・中央ロード沿いの照明関連写真だけが残る
- 史跡だけON → 中門跡・天主台跡周辺だけが残る

### 4.5 シミュ付きのみフィルタ
- 「☐ シミュ付きのみ」をチェック → シミュレーション画像が登録済みの写真ピンのみ表示
- 中間打合せでBefore/After議論用に絞り込むときに便利

### 4.6 検索する
- サイドリスト上部の 🔍 検索ボックスに入力
- ファイル名・場所・業務関連ポイント・被写体テキストで部分一致絞込
- 例：「中門跡」と入力 → 中門跡周辺の写真のみ表示
- 例：「桜山」と入力 → 桜山が写った写真（IMG_0110周辺）

### 4.7 地図切替
- 右上のレイヤコントロールで以下を切替：
  - **OpenStreetMap**（デフォルト）：道路・施設の情報量多
  - **地理院タイル（淡色）**：色がやさしく、ピンが見やすい
  - **地理院タイル（標準）**：地形・等高線あり、自然地形を見るときに

### 4.8 サイドリストから写真を見る
1. サイドリスト項目をクリック
2. 地図がその位置にズームインし、ポップアップが自動で開く
3. シミュ付きの写真は **SIM N** バッジで一目で分かる

### 4.9 サイドリストの開閉
- ヘッダの **☰ 一覧** ボタンで開閉
- 地図を広く使いたいときは閉じる
- スマホでは初期状態で閉じている

## 5. スマホでの利用

### 5.1 表示
- ヘッダの一部情報が省略されコンパクトに
- サイドリストは画面左から**ドロワー**として展開（☰ボタンで開閉）
- ピン・ポップアップ・モーダルはタッチ操作で動作

## 6. シミュレーション画像の追加について

新しい写真にシミュレーションを追加するワークフローは別ドキュメント [DESIGN.md](DESIGN.md) のセクション「8.4 シミュレーションの追加」を参照。

現時点での実装：IMG_0009（駅前広場街路灯）に A/B/C 3案を実装済み。今後 IMG_0089（中門跡）／IMG_0110（天主台跡）等に展開予定。

## 7. トラブルシューティング

| 症状 | 対処 |
|------|------|
| 地図が表示されない／ピンが出ない | ブラウザの再読込（Ctrl+F5）。それでもダメならブラウザコンソール（F12 → Console）でエラーを確認 |
| 写真が表示されない | ネット接続を確認。GitHub Pages がメンテ中の可能性も（数分後に再試行） |
| 古い表示のまま更新されない | **Ctrl + F5**（ハードリロード）またはシークレットウィンドウで開く。ブラウザキャッシュが古いコードを保持している可能性 |
| ポップアップが切れる／長すぎる | 一部のシミュ案テキストが長い場合あり。サムネクリックでモーダル拡大して全文確認 |
| サイドリストが見えない | 画面右上の ☰ 一覧 ボタンで開閉 |
| 検索しても何も出ない | 入力テキストの全角／半角・空白・誤字を確認。3〜4文字の短いキーワードを試す |
| ピンの位置が実際とずれている | 場所辞書の座標が概算のため。緑ピン（confirmed）でもズレることあり。校正は `data/location_dict.json` の lat/lon を編集 |
